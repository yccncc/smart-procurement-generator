# 智慧採購文件自動生成系統 (Smart Procurement Generator) - 模擬原型

本專案為全國政府機關通用的「智慧採購文件自動生成系統」前端原型（Prototype）。系統整合了 **2026 年中華民國政府採購法標準**（公告金額 150 萬元新制）與自動法規診斷、多文件即時勾稽連動技術，以「極簡、引導式問卷」協助基層承辦人員快速、合規地生成政府招標文件。

## 核心亮點功能說明

1. **極簡引導式問卷 (Wizard UI)：**
   - 採分步引導式卡片，拆分為「採購類別案名」、「採購金額」、「風險與保險」、「履約期限」四個核心步驟，避免一次面對大量複雜表單欄位。
   - 整合「學長姐經典範本一鍵帶入」，讓承辦人可快速加載典型範本（如大型活動、機關清潔、系統維護等預設配置）。

2. **2026 採購法智慧診斷引擎 (Core Law Engine)：**
   - **級距判定：** 自動依據「本案預算 + 後續擴充」之總金額，即時運算其金額級距（小額採購、未達公告金額、公告金額以上、查核金額以上等）。
   - **錯誤態樣防錯（守門員）：**
     - **規避公告金額招標風險（錯誤態樣二）：** 當「預算 + 後拓」總額已達公告金額 150 萬，但「本案預算金額」設定低於 150 萬時，系統會立即警告有分批辦理以規避法規之風險。
     - **履約期限合理性（錯誤態樣十）：** 當總採購金額達公告金額以上，但履約期限天數過短（如小於 15 天）時，系統會跳出警告提示承辦人履約期限不合理。
     - **工程保險合規性：** 工程採購若勾選免收保險，將發出安全警示。
   - **解釋函自動匹配：** 系統會針對案名關鍵字（如「清潔」、「系統」、「活動」、「後續擴充」）在右側即時展示相關之工程會解釋函字號與辦理規範。

3. **雙軌條文疊加與即時勾稽：**
   - 右側預覽面板即時反映左側表單輸入之案名、金額、保險與履約期限，確保《投標須知》、《契約書草案》、《需求說明書》跨文件完全一致。
   - 視覺上高亮標示：
     - **全國母軌（藍色邊框）：** 工程會標準契約及須知範本條文。
     - **機關子軌（橘色邊框）：** 機關客製特別條款。
     - **即時勾稽變數（綠色背景）：** 承辦人所填寫並即時帶入的變數，並帶有呼吸燈閃爍動畫。

4. **一鍵打包下載 Word 檔：**
   - 整合前端 Blob 生成技術，無須任何後端伺服器，保證政府採購文件機密不外洩。點擊即可直接下載排版精美且可直接用 MS Word 開啟的 `投標須知.doc` 及 `契約書草案.doc`。

## 如何運行與測試

### 方式 A：直接雙擊開啟 (最推薦，無須安裝)
1. 進入目錄 `C:\Users\YCC\.gemini\antigravity\scratch\smart-procurement-generator/`。
2. 用滑鼠雙擊 `index.html`，即可使用任意主流瀏覽器（Chrome, Edge, Safari, Firefox）開啟並進行測試。

### 方式 B：使用簡易 HTTP 本地伺服器開啟
如果您有安裝 Python，可在該目錄下執行以下指令開啟本地伺服器：
```bash
python -m http.server 8000
```
然後在瀏覽器中輸入 `http://localhost:8000` 即可訪問。

## 原型操作驗證建議

- **測試經典範本載入：**
  在 Step 1 點擊 **「學長姐經典範本一鍵帶入」** 下方的任何一個標籤（例如「115年度機關大樓資訊系統維護與資安防護委外服務案」），觀察系統是否一秒內將金額、保險、天數全部預填完畢，且右側條文及診斷警示立刻連動。
- **測試採購金額級距判定與防規避警告：**
  在 Step 2，嘗試將「本案預算」填寫為 `1,400,000` 元，「後續擴充金額」填寫為 `200,000` 元。觀察右側「法規守門員」是否立即跳出黃色警示：**「分批辦理與規避法規警示（錯誤態樣二）」**。
- **測試履約期限警告：**
  在 Step 2 將預算設為 `2,000,000` 元，然後到 Step 4 選擇「履約天數制」，輸入天數為 `10` 天。觀察右側是否立即出現警示：**「履約期限合理性診斷異常（錯誤態樣十）」**。
- **測試 Word 下載：**
  在右下角點擊 **「一鍵打包招標公文 (.docx)」**，確認瀏覽器是否自動下載兩個對應標案名稱的 `.doc` 文件，並可用 Microsoft Word 正常開啟、編輯。
